Tech Stack

Front End Languages:

  • HTML5 & CSS3 - I keep up to date with best practices, and am always looking for new creative ways of maximizing the Front End experience while understanding limitations of backwards compatibility. I strive to maintain "empathetic markup" so anyone can jump right in and understand what is going on.

  • Javascript & Jquery - I Know... How to write my own custom JS solutions. Why a plugin might make more sense. And when to hack a plugin or when to cut bait and go back to the drawing board.

  • SASS & Jeet & Jekyll- SASS is so powerful and an effective way to promote reusability and maintainability. Currently I have been using the Jeet grid system for creating responsive layouts minus all the bloat of Bootstrap or Foundation. Jeet runs on SASS, so for front-end only projects, I have been using the Jekyll static site generator to power the SASS dependency of Jeet, and spin up high fidelity prototypes very quickly. Also Jekyll's architecte resembles that of a Rails project so going from prototype to development is relatively painless.

  • Backbone & Node - Typically I have used Backbone to help organize content specific javascript. And Node as a service to interface with APIs and third party data structures.

Back End Languages:

  • Ruby on Rails - This is my strongest backend programming language. I have built robust Content Management Systems using polymorphic models, Google Maps API integration, custom social media scapers/engines.

  • PHP & Wordpress- I have coded in PHP to build a robust backend in Wordpress to allow client service teams to manage content and effortlessly create responsive newsletters. While working in PHP was a learning experience and I was able to achieve what I originally intended, Ruby seems to be born out of the ashes of PHP and solves a lot problems with much less effort.

  • Servers & Databases- I have worked with Linux servers running some combination of Apache or Puma. Typically working with staging and production environments, and deploying code updates through Capistrano. More recently I have been exploring out the box solutions with either Docker Containers, or Amazons Elastic Beanstalk. My database of choice is MYSQL given its native compatibility with Rails Projects.
